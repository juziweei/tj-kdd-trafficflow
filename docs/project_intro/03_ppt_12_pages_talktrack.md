# 12 页 PPT 讲解稿（10-15 页可裁剪）

> 每页建议讲 45~75 秒，总时长约 12~15 分钟。

## Page 1 题目与结论
- 标题：`KDD Cup 2017 收费站流量预测：从可跑到可信`
- 核心句：在防泄漏与可复现约束下，主线达到 in-split 14.95、outer 17.60。
- 讲解要点：先给结果，再给方法，再给证据。
- 建议图：一行 TL;DR 卡片（任务/约束/结果）。

## Page 2 问题定义与业务价值
- 核心句：预测 20 分钟粒度车流，直接服务运营调度。
- 讲解要点：对象、指标、业务场景。
- 建议图：任务示意图 `(tollgate, direction, time_window) -> volume`。

## Page 3 三条硬约束
- 核心句：时间切分、run_id、schema 校验是本项目不可破坏底线。
- 讲解要点：为什么这些约束比模型更先确定。
- 建议图：红线清单 + 流程闸门图。

## Page 4 核心挑战
- 核心句：非平稳、误差传播、切片异质性决定了必须分阶段设计。
- 讲解要点：挑战与失败风险的对应关系。
- 建议图：三块挑战卡片。

## Page 5 路线图
- 核心句：`baseline -> gbdt -> strong backbone -> fusion -> post-process`。
- 讲解要点：每阶段只解决一个核心矛盾。
- 建议图：阶段路线箭头图。

## Page 6 Baseline 与 GBDT 阶段
- 核心句：先打通可信流程，再增强非线性能力。
- 讲解要点：baseline 的作用不是高分，是可比较地基。
- 建议图：两阶段对照表（目标/方法/结论）。

## Page 7 Strong Backbone v6 主线
- 核心句：混合架构解决单模型难兼顾的问题。
- 讲解要点：线性底盘 + 双专家 + 动态融合 + 校正。
- 建议图：主干架构框图。

## Page 8 13 轮受控迭代
- 核心句：不是“调参碰运气”，是按验收门槛筛选。
- 讲解要点：为什么部分更低分候选被拒绝。
- 建议图：13 轮折线 + pass/fail 标注。

## Page 9 结果证据
- 核心句：in-split 14.95 是稳健结果，不是口径漂移产物。
- 讲解要点：overall + rolling + 切片联合判断。
- 建议图：结果表 + 关键切片条形图。

## Page 10 Outer 泛化与固定融合
- 核心句：承认泛化差距，用低自由度策略稳步改进到 17.6014。
- 讲解要点：为何拒绝高自由度路由，优先固定融合。
- 建议图：单主线 vs 固定融合对比图。

## Page 11 风险与失败复盘
- 核心句：失败不是噪音，是收敛路径的重要证据。
- 讲解要点：举 2~3 个失败案例和对应决策。
- 建议图：失败案例表（尝试/结果/结论）。

## Page 12 可复现闭环与下一步
- 核心句：run 级证据链让结果可回放、可审计、可协作。
- 讲解要点：下一步聚焦 outer 与长 horizon 稳健性。
- 建议图：`run_id -> config -> outputs -> check` 闭环图。

## 演讲注意事项（防讲崩）
- 每页只讲一个主句，不要一页塞三层信息。

- 主动讲风险，评委反而更信任你的结果。
